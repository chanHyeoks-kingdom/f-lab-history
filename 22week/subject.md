## 1. 토큰 기반 인증(JWT)에 대해 설명하시오
- JWT 토큰은 놀이공원 입장권 같은겁니다. 우리 웹사이트를 구경하기 위해 필요한 디지털 입장권이라고 이해하면 좋습니다.
- 로그인을 할 때 서버는 JWT라는 입장권을 발급합니다. 유저는 이 입장권을 통해 우리 서비스에 접근해 여러가지 일을 할 수 있습니다.
- 단, 이 입장권 정보를 도난당하게 되면 해당 사용자에게 인가된 모든 작업을 수행할 수 있는 셈이여서 유의해야한다.

## 2. oauth2 인증 작동방식에 대해 설명하시오

#### [a. 보안 관련]
> a1. OAuth에서 리디렉션 URI의 보안을 어떻게 확보 가능한지
- 리디렉션이란건 결국 사용자가 인증을 거친 후 데이터가 반환되는 주소입니다.
- 이 주소가 타당한지 확인을 통해 URI를 조작하는 공격을 방지하기 위한 조치를 취해야합니다.
```
리다이렉션 URI는 사전에 등록된 주소만 허용되어야 합니다.
이를 통해, 인증 후 토큰이 조기 신뢰할 수 있는 사전 정의된 주소로만 전송되도록 보장합니다.
와일드카드 사용을 피하고 정확한 주소를 명시해 보안을 강화할 수 있습니다.
```

> a2. OAuth 토큰을 안전하게 저장하는 방법은 무엇인가요?
- 액세스 토큰은 민감 정보를 포함하고 있으며,
- 이를 안전하게 저장하는 방법에 대한 질문은 필수적입니다.
```
서버 측에 토큰을 저장하는 것이 가장 안전합니다.
클라이언트 측 저장 시에는 액세스 토큰 대신 리프레시 토큰만을 저장하고,
가능하면 보안이 강화된 저장소(예: HTTPS 쿠키, 암호화된 데이터베이스)를 사용해야 합니다.
```


> a3. OAuth 과정에서 CSRF(사이트 간 요청 위조) 공격을 어떻게 방지할 수 있나요?
- CSRF는 사용자가 자신의 의지와는 무관하게 공격자가 원하는 행동을 하도록 만드는 공격입니다.
- OAuth를 사용할 때 이러한 공격을 방지하기 위한 방법을 논의할 필요가 있습니다.
```
CSRF 공격을 방지하기 위해, 인증 요청에는 상태 매개변수(state parameter)를 포함시켜야 합니다.
이 상태 매개변수는 요청이 시작될 때 생성되며, 인증이 완료된 후 서버로 반환되어 요청의 유효성을 검증하는 데 사용됩니다.
```

```

> b2. 다중 클라이언트 환경에서 OAuth를 어떻게 효과적으로 관리할 수 있나요?

- 각 클라이언트에 적합한 인증 흐름을 어떻게 구현하고 관리할 것인지에 대한 질문입니다.
```
각 클라이언트 유형(웹, 모바일, 데스크톱)에 맞는 인증 흐름을 구현해야 합니다.
이는 클라이언트가 실행되는 환경의 특성과 보안 요구를 충족시키는 데 중요합니다.
또한, 모든 클라이언트가 중앙 인증 서버와 효과적으로 통신할 수 있도록 관리해야 합니다.
```

> b3. OAuth의 리프레시 토큰은 어떻게 안전하게 재발급하나요?
- 리프레시 토큰은 액세스 토큰을 재발급 받기 위해 사용됩니다.

동의 과정은 사용자에게 충분한 정보를 제공하여 정보에 기반한 결정을 내릴 수 있도록 해야 합니다.

이는 사용자 인터페이스를 통해 쉽게 접근하고 실행할 수 있어야 하며,
동의 철회가 이루어진 경우 관련 데이터 접근 권한을 즉시 중지해야 합니다.
```

## 3. SSO(Single Sign On)에 대해 설명하시오

#### [a. 보안 관련 주제]

> a1. SSO 시스템의 보안을 어떻게 강화할 수 있나요?
- SSO 시스템이 중앙 집중화된 인증을 제공하기 때문에, 보안 강화는 필수적입니다. 
- 어떤 종류의 보안 프로토콜과 조치가 이를 지원하는지 질문할 수 있습니다.
```
SSO 시스템의 보안을 강화하기 위해 다중 요소 인증(Multi-Factor Authentication, MFA)을 구현하는 것이 중요합니다.
또한, 정기적인 보안 감사 및 취약점 평가를 실시하여 시스템의 취약점을 조기에 발견하고 수정해야 합니다.
모든 통신은 HTTPS를 통해 암호화되어야 하며, 안전한 토큰 관리 전략이 필요합니다.
```

> a2. SSO 도입 시 사용자 데이터 보호는 어떻게 처리되나요?
- 사용자 데이터의 안전한 처리와 저장을 보장하기 위한 방법이 중요합니다. 
- 사용자 정보가 어떻게 암호화되고 관리되는지에 대한 질문입니다.
```

SSO 시스템의 백업 및 복구 전략을 준비하기 위해 정기적인 백업을 수행하고, 재해 복구 테스트를 통해 복구 프로세스를 검증해야 합니다.
또한, 백업 데이터는 안전한 위치에 보관하며, 암호화하여 외부의 접근으로부터 보호해야 합니다.

인증은 사용자가 누구인지 확인하는 과정이며, 권한 부여는 사용자가 수행할 수 있는 작업을 결정하는 과정입니다.
SSO 시스템에서 이 두 과정을 구분하기 위해 인증 후에 사용자의 역할과 권한을 관리하는 별도의 권한 관리 시스템을 구현해야 합니다.
```

## 4. 객체 직렬화(Serialization)와 역직렬화(Deserialization)에 대해 설명하시오
#### [a.보안 관련 주제]
> a1. 직렬화된 데이터의 보안 취약점은 무엇인가요?
- 직렬화된 데이터가 어떻게 보안 위협이 될 수 있는지,
- 특히 외부에서 접근 가능할 때 어떤 위험 요소들이 있는지에 대한 질문입니다.
```
직렬화된 데이터는 외부에 노출될 경우 악용될 수 있습니다.
특히, 역직렬화 과정에서 악의적인 데이터가 실행되어 시스템을 손상시킬 수 있는 취약점이 발생할 수 있습니다.
따라서, 직렬화 데이터는 보안적으로 취약한 점을 인지하고 적절한 보호 조치를 취해야 합니다.
```

> a2. 직렬화 과정에서 데이터 조작을 방지하기 위한 방법은 무엇인가요?
- 데이터가 직렬화되어 전송되는 과정에서 조작되지 않도록 보호하기 위한 기술적 조치는 무엇인지에 대한 질문입니다.
```
데이터 조작을 방지하기 위해서는 직렬화된 데이터에 디지털 서명을 추가하여 데이터의 무결성을 검증할 수 있습니다.
이 방법은 데이터가 변조되지 않았음을 보증하고, 신뢰할 수 있는 출처에서만 데이터가 왔음을 확신할 수 있게 해줍니다.
```

> a3. 직렬화 데이터를 안전하게 전송하는 방법은 무엇인가요?
- 직렬화된 데이터를 네트워크를 통해 전송할 때 적용할 수 있는 보안 프로토콜과 방법에 대한 질문입니다.
```
직렬화된 데이터를 안전하게 전송하기 위해, 데이터를 전송하기 전에 암호화하는 것이 중요합니다.
SSL/TLS와 같은 안전한 통신 프로토콜을 사용하여 데이터를 암호화하고, 이를 통해 데이터의 기밀성과 무결성을 보장합니다.
```

#### [b. 성능 및 최적화 관련 주제]
> b1. 직렬화 프로세스가 시스템 성능에 미치는 영향은 무엇인가요?
- 직렬화가 시스템 리소스에 미치는 영향과 성능 저하를 최소화하기 위한 전략은 무엇인지에 대한 질문입니다.
```
직렬화 프로세스는 시스템의 CPU와 메모리 리소스를 상당량 사용할 수 있습니다.
특히, 큰 객체나 복잡한 데이터 구조를 직렬화할 때 성능 저하가 발생할 수 있습니다.
이를 최소화하기 위해 데이터 구조를 단순화하고, 필요한 데이터만 직렬화하는 것이 중요합니다.
```

> b2. 다양한 직렬화 형식(XML, JSON, 바이너리 등)의 장단점은 무엇인가요?
- 각기 다른 직렬화 형식들이 상황에 따라 어떻게 유리하거나 불리할 수 있는지에 대한 질문입니다.
```
XML: 장점은 읽기 쉽고, 표준화된 형식이라는 것입니다. 단점은 데이터 크기가 상대적으로 크고, 파싱 속도가 느릴 수 있다는 점입니다.
JSON: 장점은 경량이고, 웹 환경에서 널리 사용되며, 파싱이 빠르다는 점입니다. 단점은 복잡한 데이터 구조를 표현하는 데는 한계가 있을 수 있다는 것입니다.
바이너리: 장점은 처리 속도가 매우 빠르고, 데이터 크기가 작다는 점입니다. 단점은 가독성이 없으며, 플랫폼 간 호환성 문제가 발생할 수 있다는 것입니다.
```

#### [c. 데이터 관리 및 호환성 주제]
> c1. 직렬화 데이터의 버전 관리는 어떻게 이루어지나요?
- 시스템이 업그레이드되거나 변경될 때, 이전에 직렬화된 데이터와의 호환성을 유지하기 위한 방법에 대한 질문입니다.
```
버전 관리를 통해 직렬화된 데이터의 호환성을 유지하려면, 객체의 버전 번호를 데이터에 포함시키고,
역직렬화할 때 이 번호를 확인하여 호환 가능한 버전의 코드로 처리하는 로직을 구현해야 합니다.
이는 데이터 구조가 시간에 따라 변경될 때 이전 버전과의 호환성을 보장합니다.
```

> c2. 다양한 프로그래밍 언어 간의 직렬화 데이터 호환성은 어떻게 보장할 수 있나요?
- 서로 다른 언어나 플랫폼에서 작동하는 시스템들 사이에 데이터를 교환할 때 발생할 수 있는 문제를 해결하기 위한 전략에 대한 질문입니다.
```
언어 간 호환성을 보장하기 위해 표준화된 데이터 포맷(예: JSON, XML)을 사용하는 것이 중요합니다.
이 포맷들은 대부분의 프로그래밍 언어에서 지원되므로, 서로 다른 시스템 간의 데이터 교환이 용이해집니다.
```
  
#### d. 법적 및 규정 준수 주제
> d1. 직렬화 데이터의 저장 및 처리에서 규정 준수는 어떻게 이루어지나요?

```
데이터 보호 규정 준수를 위해, 모든 직렬화된 데이터는 법적 요구 사항에 따라 처리되어야 합니다.
예를 들어 GDPR에서는 개인 데이터의 안전한 처리를 요구하므로, 데이터 암호화, 접근 제어, 사용자 동의 수집 등의 조치를 취해야 합니다.
```
