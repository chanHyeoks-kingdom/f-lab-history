#### 이번주 학습하면서 막막했던 부분
> ##### [동시성이라는 개념에 대한 위압감]
```
- 나는 학습을 할 때 전체적인 틀을 이해 하고 접근해야 불안감을 덜 느끼는 거 같은데 동시성 개념 자체가 OS, DB, 프로그래밍, 네트워크 등.. 광범위한 분야인 느낌이어서 해당 개념들을 공부할 때 자꾸 막막한 느낌을 받았던 거 같다.
- 그러다보니 금주 주제인 Java Synchronized Collection, Concurrent Collection 자체에 대해 집중하지 못하고 전체적인 틀을 파악하려는데 시간을 쏟았는데, 필요한 과정이긴 하나 지금 해야 했던 일은 아니였음을 깨달았다.
- 뒤늦게 해당 키워드들에 대해 접근했을 때 내부 코드를 이해하는데 어려움을 느꼈고 CAS나 세그먼트, 락 프리, 논 블록킹 등 키워드 등으로 학습 범위가 넓어지니 뭘 먼저 봐야하나 고민하는 시간의 비중이 높아졌다.
- 진도가 정체되니 불안해서 다른 주제를 먼저 보게 됐고 마무리 할 때 즈음 적당히 알아보게 됐다.
- 결국 깊게 이해하지도 못했고 그렇다고 기능적인 부분을 잘 알게 된 것도 아닌 애매한 상태로 학습이 마무리 됐다.
```

#### 이번 주에 멘토링을 진행하며 스스로에게 '좋았던 점'과 '보완하고 싶은 점'을 적어주세요.
> ##### [좋았던 점]
```
1. 나는 스스로 생각하기에 인성이 좋다고 생각해서 그냥 솔직하게만 말하면 인성 면접은 만사 오케이 일 줄 알았는데 오늘 멘토링을 통해 그게 아니라는 사실을 깨달아서 감사한 하루였다.
2. 문제 사례를 직접 코드로 해결해보는 과정을 통해 thread Safety 관련 학습을 진행해보자는 결정을 할 수 있게 되서 좋았다. 그 동안도 비슷하게 진행하고 있었는데 이 방법을 더 적극적으로 적용해봐도 괜찮을 거 같다는 느낌을 받았다.
3. HTTP Method나 Status Code 등은 단순히 권고안 정도의 개념이라는 것에 대해 확신이 없었는데, 멘토링 과정에서 해당 인사이트를 얻을 수 있어 좋았다.
4. 어떤 개념이든 '불편함' 뒤에 '개선'을 목적으로 나온다는 원리를 배울 수 있어 감사했다.
```

> ##### [보완하고 싶은 점]
```
1. 대부분의 여유 시간은 공부 하는데 사용하고 있어서 시간이 부족하진 않지만 인성면접 답변 준비하는 시간이 왠지 아까워서 미루고 있었다. 오늘 과정을 통해 시간 내서 준비해야겠다고 생각했다.
2. MVC나 서블릿 프로그래밍은 전반적으로 한 번 코딩으로 훑어본 적이 있어서 시간적 여유가 있었던 거 같다. 멘토님이 학습 분량에 대해서 말씀하실 때 이걸 제대로 말씀 못드렸다.
3. 멘토님께 동시성 관련 내용 학습 방법에 대해 질문드렸는데, 오늘 칭찬 받은게 너무 강렬한 자극이어서 멘토님 답변 주시는데 그 생각하느라 답변 내용을 못들었다. 칭찬 받을 때 침착할 필요가 있을 거 같다.
4. HashMap이나 Synchronized Collection이나 Concurrent Collection에 대해 다시 잘 정리해봐야 할 거 같다.
```
