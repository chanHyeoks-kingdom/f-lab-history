# 이번주 멘토링 회고


#### 기술 질문
> ##### [금주 받은 질]
```
1. 객체지향 개념을 대해야 하는 방법에 대해 질문을 통해 이해시켜주셨음
  - 팩토리 패턴, 생성자등의 설명 과정에서 '이게 더 현실성 있는지'를 고민하며 접근해보는 것의 중요성을 알려주셨다.

2. 중요한 개념에 대한 핵심을 이해할 수 있도록 주신 질문들
  - 팩토리 패턴과 팩토리 메서드 패턴 설명 과정에서 '문자열'을 사용했을 때 생기는 문제에 대해 이해시켜주셨다. 
  - DI와 스프링 컨테이너가 필요한 이유에 대해서 객체 직접 생성, 생성자 혹은 필드 주입, DI, DI Container의 순으로 유도 질문을 해주셨다.

3. 더 깊게 공부해보면 좋을 방향성을 알려주시는 질문들
  - 내가 작성한 simple factory에서 아이폰, 삼성폰을 생성하는 책임을 가져가는게 맞는가?
  - 생성자 주입을 사용하면 얻을 수 있는 이득 (자동차와 엔진 비유)
  - 세터주입과 생성자 주입이 어떤 차이가 있을까요? (최대한 실생활에 근거해서)
  - 객체를 만듦에 있어서 생성자에 들어가는 매개변수들은 필수일까요 선택일까요?
  - 서로 다른 애플리케이션에서 공통된 관심사는 어떻게 공유할 수 있을까요?
  - 싱글톤은 JVM에 하나만 생성되서 공유되는데 이 떄 어떤 문제가 발생할 수 있을까요?

4. 그 외 프로젝트 관련 질문들을 해주셨다.
```





#### 회고
> ##### [막막했던 부분]
```
[요점과 핵심을 파악 하지 못함]
계속 고쳐보겠다고 고민하는 문제인데, 질문의 요점을 파악하고 빠르게 관련된 키워드를 파악하는 능력이 부족한 거 같다.
자꾸 헛도는 시간들이 있었고, 코드를 직접 짜는 비중을 처음으로 많이 늘려본 주여서 그런지 유독 중요한 키워드를 잘 정리하지 못했다.
```

> ##### [좋았던 점과 보완하고 싶은 점]
```
[좋았던 점]
  - 사실 그 동안 코드 자체를 짜볼 일이 많이 없었다. 대부분은 알고리즘 풀 때나 강의를 따라치는 수준이였는데, 금주 과제를 통해 실제로 생각하면서 코딩을 하는게 진짜 재밌는 일이구나라는 걸 깨달아서 너무 감사한 한 주였다.
  - 코드 위주로 공부하니까 너무 재밌어서 멘토링 날이 다가온줄도 모를 정도로 재밌게 보냈다.
  - 개발자로서 주관이 조금씩 잡혀가고 있는 느낌을 받고 있는데 덕분에 개발자로서 어떻게 살고 싶은지 소신이 생기기 시작했다.

[보완하고 싶은 점]
  - '어떻게 써야하는 지'에만 너무 몰두하다 보니 전체적인 흐름 파악에 소홀했음을 깨달았고 이론과 프로그래밍에 적절한 비율을 조정하는 법을 고민해보자는 결론을 얻었다.
  - '계획을 꼼꼼히 하는 능력'이 부족하다는인사이트를 얻었다. 프로젝트 들어가면 이 특징이 더 도드라질 거 같아서 더욱 정신 차리고 꼼꼼히 일정을 계획해야겠는 생각을 했다.

```
